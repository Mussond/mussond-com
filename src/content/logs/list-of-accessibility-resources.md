## Beginner courses

1. [UX Foundations: Accessibility](https://www.linkedin.com/learning/ux-foundations-accessibility) — Derek Featherstone, LinkedIn Learning. Good intro to accessibility, users and the barriers people face.
2. [Web Accessibility Tutorials](https://www.w3.org/WAI/tutorials/) — W3C WAI. Helpful examples and detail to build on the foundations course.
3. [Accessibility for Web Design](https://www.linkedin.com/learning/accessibility-for-web-design) — Derek Featherstone, LinkedIn Learning. Granular examples and how-tos. A solid extension of the foundations course.
4. [A11ycasts with Rob Dodson](https://www.youtube.com/playlist?list=PLNYkxOF6rcICWx0C9LVWWVqvHlYJyqw7g) — YouTube. Granular examples with a code and screen reader focus.

## WCAG quick reference

- [WCAG2 Quick Reference](https://3pha.com/wcag2/) — all WCAG success criteria on one page. Useful overview.

## Checklists and audits

1. [WebAIM WCAG 2 Checklist](https://webaim.org/standards/wcag/checklist) — comprehensive summary of WCAG, good for intermediate level.
2. [WCAG, but in language I can understand](https://www.tempertemper.net/blog/wcag-but-in-language-i-can-understand) — tempertemper. Simple plain-English copy for each criterion. Good for beginners.
3. [Giving a Damn About Accessibility](https://www.accessibility.uxdesign.cc/) — useful article/PDF covering user needs and organisational challenges.
4. [The A11Y Project Checklist](https://www.a11yproject.com/checklist/) — written in plain English, grouped by category.

Other checklists:

- [Accessibility Not-Checklist](https://not-checklist.intopia.digital/) — Intopia. Guidance grouped by topic, role and best practice.
- [MagentaA11y Web Checklist](https://www.magentaa11y.com/web/) — detailed accessibility acceptance criteria by component.
- [Web Accessibility Checklist](https://webaccessibilitychecklist.com/) — basic checklist in plain English.
- [WCAG Testing Checklist](https://www.accessguide.io/wcag-testing-checklist) — Accessguide. Grouped by automated, semi-automated and manual testing.

## Cognitive disabilities

1. [Accessible Numbers](https://accessiblenumbers.com/) — guidelines for presenting numbers and data clearly, covering low numeracy and dyscalculia.
2. [An Introductory Guide to Understanding Cognitive Disabilities](https://www.deque.com/blog/an-introductory-guide-to-understanding-cognitive-disabilities/) — Deque.

## Accessibility programme

1. [How to document the screen reader user experience](https://bbc.github.io/accessibility-news-and-you/guides/screen-reader-ux.html) — BBC. A thorough example of documenting an accessible UX.
2. [How Our Organisation Improved Web Accessibility](https://www.smashingmagazine.com/2022/08/organization-improved-web-accessibility-case-study/) — Smashing Magazine case study covering the most common accessibility issues.
3. [A real-world strategy for building accessible products](https://medium.com/@uxeng/a-real-world-strategy-for-building-accessible-products-210c7cbe6ff) — Heenesh Mistry.

## Web references

1. [MDN HTML Docs](https://developer.mozilla.org/en-US/docs/Web/HTML) — the authoritative reference for HTML specs.
2. [ARIA Authoring Practices Guide](https://www.w3.org/WAI/ARIA/apg/) — W3C. ARIA specs and worked examples.
3. [WAI-ARIA 1.1 Cheat Sheet](https://cheatography.com/jreiche/cheat-sheets/wai-aria-1-1/) — roles, attributes, global states, properties and landmarks at a glance.
4. [Accessibility Support](https://a11ysupport.io/) — community-driven tests showing how component builds behave with screen readers.
